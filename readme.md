Дошка оголошень групи — Лабораторна робота №5 (Уразливості і захист)

Цей проєкт є фінальною ітерацією навчального Fullstack-вебдодатка. Головною метою цієї версії є перехід до використання реальної бази даних та впровадження базових практик безпечної розробки (Secure Coding) для захисту від найпоширеніших веб-уразливостей.

Реалізовані сценарії захисту

Додаток успішно захищено від наступних атак (згідно з вимогами лабораторної роботи):

1. SQL Injection (SQLi)
   * Проблема: Можливість зламати структуру SQL-запиту через введення спецсимволів (наприклад, `' OR 1=1 --`) у поле пошуку.
   * Захист: Використання виключно параметризованих запитів (`SELECT * FROM ads WHERE title LIKE ?`) через драйвер `sqlite`. Дані користувача більше не конкатенуються безпосередньо в рядок запиту.

2. Cross-Site Scripting (XSS)
   * Проблема: Ризик виконання шкідливого JavaScript-коду (Stored XSS) при відображенні оголошень з тегами `<script>` або `<img onerror="...">`.
   * Захист: Повна відмова від використання `innerHTML`. Рендеринг даних на клієнті виконується виключно через DOM API (`document.createElement` та `textContent`), що гарантує інтерпретацію будь-якого вводу як звичайного тексту.

3. Broken Access Control (IDOR)
   * Проблема: Можливість користувача видалити чуже оголошення, просто відправивши DELETE-запит із чужим ID.
   * Захист: Впроваджено демо-ідентифікацію через заголовок `X-Demo-UserId`. На стороні сервера (Backend) додано сувору перевірку: сервер порівнює `ownerUserId` оголошення в БД із поточним ID користувача, який робить запит. При неспівпадінні повертається статус `403 Forbidden`.

4. Security Misconfiguration
   * Проблема: Витік чутливої інформації (stack trace) при помилках та відсутність базових захисних HTTP-заголовків.
   * Захист: * Додано заголовки безпеки (`X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, `Referrer-Policy: no-referrer`).
     * Налаштовано гнучкий, але обмежений **CORS** (дозволено лише локальні origin).
     * Централізована обробка помилок приховує системні деталі бази даних від клієнта.

Інструкція із запуску

1. Запуск бекенду (Сервер + База даних)
У першому терміналі перейдіть у папку `backend` та виконайте команди:

Bash
cd backend
npm install
npx ts-node server.ts

2. Запуск фронтенду (Клієнтська частина)
У другому терміналі перейдіть у папку frontend, скомпілюйте код та запустіть сервер:

Bash
cd frontend
npx tsc
npx live-server public
